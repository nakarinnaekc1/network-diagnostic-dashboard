📦 FILES ที่จะต้องใช้:
═══════════════════════════════════════════════════════════════════

✅ app.py                 ← Flask application
✅ requirements.txt       ← Python dependencies

✅ .gitignore            ← Git ignore file
✅ README.md (optional)  ← Project description


🎯 STEP 1: สร้าง GitHub Repository
═══════════════════════════════════════════════════════════════════

ทำบน GitHub:

1. ไปที่ github.com → Click "New" เพื่อสร้าง repository
2. Repository name: network-diagnostic-dashboard
3. Description: Network Diagnostic Dashboard with AI
4. ✅ ทำให้เป็น Public (เพื่อ Render เข้าได้)
5. ✅ Add a README file
6. Create repository


📝 STEP 2: Push Files ไปยัง GitHub
═══════════════════════════════════════════════════════════════════

ในคอมพิวเตอร์ของคุณ:

A. Clone repository:
   git clone https://github.com/YOUR_USERNAME/network-diagnostic-dashboard.git
   cd network-diagnostic-dashboard

B. Copy all files ที่คุณได้:
   - app.py
   - requirements.txt
   - Procfile
   - runtime.txt
   - .gitignore
   
   (ลง folder network-diagnostic-dashboard)

C. Push ไป GitHub:
   git add .
   git commit -m "Initial commit - Network diagnostic dashboard"
   git push origin main

✅ ตรวจสอบว่า GitHub มี files ทั้งหมด


🌐 STEP 3: สร้าง Render Account & Service
═══════════════════════════════════════════════════════════════════

ไปที่ render.com:

1. Sign up (ฟรี) ด้วย GitHub account
   → Render.com → Sign up → "Continue with GitHub"
   → Authorize Render

2. เมื่อ login แล้ว → Click "+ New" → "Web Service"

3. Connect repository:
   - ค้นหา: network-diagnostic-dashboard
   - Click "Connect"

4. ตั้งค่า Web Service:

   Name: network-diagnostic-dashboard
   
   Environment: Python 3
   
   Build Command: pip install -r requirements.txt
   
   Start Command: gunicorn app:app
   
   Instance Type: Free (ได้เลย)
   
   Region: Singapore (ใกล้สุด)


⚙️ STEP 4: ตั้ง Environment Variables
═══════════════════════════════════════════════════════════════════

สำคัญ! ต้องตั้ง ANTHROPIC_API_KEY ให้ Render:

1. ใน Render Dashboard → Web Service ของคุณ
2. ไปที่ "Environment" tab
3. Click "+ Add Environment Variable"
4. 
   Key:   ANTHROPIC_API_KEY
   Value: sk-ant-your-actual-api-key-here
   
5. Click "Save"
6. Service จะ redeploy อัตโนมัติ


🚀 STEP 5: Deploy
═══════════════════════════════════════════════════════════════════

Deploy สามารถทำได้ 2 วิธี:

วิธีที่ 1: Automatic (แนะนำ)
   - Render watch GitHub repository
   - ทุกครั้ง push ไป GitHub จะ deploy อัตโนมัติ
   - ไม่ต้องทำอะไรเพิ่มเติม

วิธีที่ 2: Manual
   - Render Dashboard → Manual Deploy
   - Click "Deploy latest"


📱 STEP 6: ใช้งาน
═══════════════════════════════════════════════════════════════════

1. ตรวจสอบ Deployment status:
   - Dashboard ต้องแสดง "Live" (สีเขียว)
   - ปกติใช้เวลา 2-5 นาทีตั้งแต่ push ไป GitHub

2. ได้ Public URL:
   - Render Dashboard → Web Service
   - URL อยู่ด้านบน
   - ตัวอย่าง: https://network-diagnostic-dashboard.onrender.com

3. เปิด Browser:
   https://your-service-name.onrender.com

4. Click "▶️ เริ่มการตรวจสอบ"


⚠️ IMPORTANT NOTES:
═══════════════════════════════════════════════════════════════════

   • ครั้งแรกใช้ประมาณ 30 วินาที (cold start)
   • Disk space ไม่มาก (OK สำหรับ app นี้)

✅ Solutions:
   • Upgrade ไป Paid Plan ($7/month) ถ้าต้องการ 24/7
   • หรือให้ Render ทำงาน โดยไม่หยุด

❌ API Calls:
   • Anthropic API มี rate limits
   • ตรวจสอบ API usage ใน Anthropic dashboard

✅ Best Practice:
   • Monitor Render logs เพื่อหา errors
   • ตั้ง Alert สำหรับ errors
   • Update requirements.txt เมื่อต้องการ lib ใหม่


📊 MONITORING & DEBUGGING:
═══════════════════════════════════════════════════════════════════

ถ้า Deploy ล้มเหลว:

1. ตรวจสอบ Logs:
   Render Dashboard → Web Service → Logs tab
   
2. Common Errors:
   
   ❌ "ModuleNotFoundError"
      → requirements.txt ขาดบรรทัด
      → เพิ่มลง requirements.txt แล้ว push ใหม่
   
   ❌ "Address already in use"
      → app.py ใช้ port ไม่ถูกต้อง
      → ตรวจสอบ: port = int(os.environ.get("PORT", 5000))
   
   ❌ "API Key not configured"
      → ลืม set ANTHROPIC_API_KEY
      → ไปที่ Environment → add variable
   
   ❌ "Timeout"
      → Diagnostic ใช้เวลานาน
      → Increase timeout ใน app.py (ปัจจุบัน 30-60 วินาที OK)

🔐 SECURITY BEST PRACTICES:
═══════════════════════════════════════════════════════════════════

✅ DO:
   • เก็บ API Key ใน Environment Variable
   • ไม่ commit .env file ไป GitHub
   • ใช้ strong API key จาก Anthropic
   • Enable HTTPS (Render ทำให้อัตโนมัติ)

❌ DON'T:
   • ไม่ใส่ API Key ใน app.py
   • ไม่ share URL กับคนนอก (แม้ฟรี tier ก็มี usage cost)
   • ไม่ลืม .gitignore
