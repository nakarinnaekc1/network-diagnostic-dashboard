⚡ 5-MINUTE DEPLOYMENT:
═══════════════════════════════════════════════════════════════════

🔴 1️⃣ MINUTE 1 - GitHub Setup (1 min)

   A. Go to github.com → Sign in/Create account
   
   B. Create new repository:
      • Name: network-diagnostic-dashboard
      • Public ✅
      • Create repository
   
   C. Upload files (GitHub web interface):
      • Click "Add file" → "Upload files"
      • Drag & drop all 6 files
      • Commit changes


🔴 2️⃣ MINUTE 2 - Files Check (1 min)

   Verify in GitHub:
   ✅ app.py
   ✅ requirements.txt
   ✅ Procfile
   ✅ runtime.txt
   ✅ .gitignore
   ✅ README.md


🔴 3️⃣ MINUTE 3 - Render Connection (1 min)

   A. Go to render.com → Sign up
      • "Continue with GitHub" (easiest)
      • Authorize Render
   
   B. After login:
      • Click "+ New"
      • Select "Web Service"
   
   C. Connect GitHub:
      • Select your repository
      • Click "Connect"


🔴 4️⃣ MINUTE 4 - Configuration (1 min)

   Fill in these fields:
   
   • Name: network-diagnostic-dashboard
   • Environment: Python 3
   • Build Command: pip install -r requirements.txt
   • Start Command: gunicorn app:app
   • Instance Type: Free
   • Region: Singapore (or nearest)
   
   Click "Create Web Service"


🔴 5️⃣ MINUTE 5 - API Key Setup (1 min)

   ⚠️ CRITICAL STEP - Do this or AI won't work!
   
   A. In Render Dashboard:
      • Go to "Environment" tab
      • Click "+ Add Environment Variable"
   
   B. Fill in:
      Key:   ANTHROPIC_API_KEY
      Value: sk-ant-your-actual-api-key
   
   C. Click "Save"
   
   D. Render auto-redeploys (wait 2-5 min)

❓ TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════════

❌ Shows error on load:

   → Check Render Logs (Dashboard → Logs)
   
   Common issues:
   ✓ API Key not set → Add in Environment
   ✓ Syntax error → Check app.py
   ✓ Missing dependency → Update requirements.txt


❌ Deployment still pending after 5+ min:

   → Check build logs in Render
   → Verify all files uploaded
   → Try manual deploy: Dashboard → "Deploy latest"


❌ "ANTHROPIC_API_KEY" error:

   → Go to Environment tab
   → Add environment variable EXACTLY as:
      Key: ANTHROPIC_API_KEY
      Value: sk-ant-xxxxx
   → Service will redeploy automatically
